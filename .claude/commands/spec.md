# Consultar Spec

Lee y resume el spec relevante para: $ARGUMENTS

Buscar en este orden:
1. `specs/functional/` — para entender qué debe hacer el módulo (reglas de negocio, flujos, entidades)
2. `specs/technical/` — para entender cómo implementarlo (arquitectura, modelo de datos, convenciones)

Mostrar:
- Las reglas de negocio críticas del módulo
- Los atributos de la entidad principal
- Las validaciones requeridas
- Lo que está explícitamente fuera de alcance del MVP
